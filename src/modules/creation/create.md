# 资源表需求梳理与字段设计 （file_resource）

## 一、核心需求
1. 收集创作模块产出的图片资源，便于后续筛选、推荐和复用。
2. 支持手动添加外部图片资源，统一管理。
3. 通过来源字段区分图片的来源（如创作模块、手动上传等）。
4. 支持对图片资源的分类、标签和适用场景标记。
5. 记录图片被使用的次数，便于后续做推荐。

## 二、表字段初步设计
| 字段名         | 类型         | 说明                         | 备注           |
| -------------- | ------------ | ---------------------------- | -------------- |
| id             | number       | 主键，自增ID                 |                |
| url            | string       | 图片资源URL                  | 必填           |
| origin         | string       | 图片来源（如creation/manual）| 必填           |
| type           | string       | 图片类型/分类                | 选填           |
| tags           | string[]     | 标签                         | 选填，多个          |
| scene          | string[]       | 适用场景（如早安/晚安等）    | 选填，多个          |
| weight         | number       | 权重 ， 0-100                | 选填，默认0         |
| usageCount     | number       | 被使用次数                   | 默认0          |
| description    | string       | 图片描述                     | 选填           |
| createdAt      | Date         | 创建时间                     | 自动生成       |
| updatedAt      | Date         | 更新时间                     | 自动生成       |
| originId       | string       | 来源ID（可用于溯源）           | 选填           |

## 三、功能需求规划
1. 支持批量导入图片资源（如从创作模块批量选入）。
2. 支持按来源、标签、场景等多条件筛选图片。
3. 支持图片资源的软删除和恢复。
4. 支持图片资源的编辑（如修改标签、描述、场景等）。
5. 支持统计和推荐功能（如按usageCount推荐热门图片）。
6. 支持扩展更多来源类型和图片属性。

## 四、后续扩展建议
- 可考虑增加图片审核状态字段，支持内容安全管理。
- 可扩展为多媒体资源表，支持视频、音频等。
- 可与用户表关联，记录上传/推荐人。
- 可增加图片尺寸、格式等元数据字段。
- 可通过originId字段实现图片资源的溯源（如关联原始创作、外部来源等）。

---

> 以上为图片资源表的初步需求梳理和字段设计，后续可根据实际业务需求进一步细化和调整。
